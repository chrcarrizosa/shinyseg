Liability class
The liability class table contains the probability of observing a given phenotype conditionally on the presence of 0, 1 or 2 copies of the variant (f0, f1 and f2 columns). This is by default a single vector that would apply to all cases, but a specification dependent on sex, phenotype and age can also be made. This can be done by loading an external file through Load file or by adding new rows with ADD and filling in the necessary information in such a way that there are no overlaps between the three subsequent columns:
  • sex: male, female or both for classes that refer to males, females or both sexes, respectively.
  • phenotype: a free-text field whose values should match the phenotypes specified in the pedigree table. Note that nonaff is another phenotype whose classes must be also specified (unless there are no unaffected cases); leaving this cell empty indicates that the class apply to all phenotypes(s).
  • ages: an integer from 1 to 100, or ranges between these values (e.g. 10-20, 30-100...) that indicate the ages to which the class applies; leaving this cell empty is the same as specifying a range 1-100 (i.e. the class applies to all ages).
